object model description


main
- parse user input arguments
- initialize virtual machine

vm (virtual machine)
- main object initialize other modules
- initialize main instruction execution loop
- in execution loop command engine to execute single instruction
- handle ending sequence

memory
- contains loaded code and data segment
- initialized with help of io and header
- provadides read/write operations above memory segments

header
- with io object parse header part of binary object and run header sizes and format checks
- is owned by memory

io
- provides file operations like read/write from/to file

decoder
- provides decoding of code segment
- possible to decode instruction, address, constant or argument
- works with specific registers - during decoding is incrementing instruction pointer (counter)

registers
- contains registers, stack, instruction pointer for some specific thread

engine
- contains all instruction objects (note: polymorphism demonstration)
- handle single instruction run
- handle access to critical sections based on instruction type with help of threading model

instruction, instructions_*
- contain instruction logic
- works with decoder, memory, io, threading model

threading model
- contains logic related to handling of threads
- used by thread related instructions (createThread, joinThread, lock, unlock)
- contains mutexes for interpreted program run and mutex for access to critical section access in vm
