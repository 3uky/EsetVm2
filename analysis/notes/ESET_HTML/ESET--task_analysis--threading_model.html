threading model


Threading model

each thread has own
- registers (copied from parent)
- stack (empty)
- instruction pointer (initialized with createThread argument value)

main thread
- has to join all initialized threads before exits (don't rely on joinThread i. in easm program)

thread synchronization
- two level of synchronization - defined by binary + critical sections in virtual machine
- threads could be synchronized with lock, unlock instruction based on easm/evm program
- threads has to be synchonized also in virtual machine
→ threads could overwrite e.g.: arguments, addresses, etc. to each other
→ several threads cannot write into file at once
→ instructions with KS have to be synchronized
